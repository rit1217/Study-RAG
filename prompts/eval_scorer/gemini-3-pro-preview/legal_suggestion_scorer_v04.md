# รูปแบบข้อมูลเฉลย
เฉลย (EXPECTED) อยู่ในรูปแบบ Markdown ที่แบ่งเป็นหัวข้อ:
- `## 1. มาตรากฎหมายที่เกี่ยวข้อง` — มาตรากฎหมายและคำพิพากษาศาลฎีกาที่เกี่ยวข้อง
- `## 2. คำวินิจฉัย` — คำวินิจฉัยที่ถูกต้อง
- `## 3. ข้อสรุปและข้อเสนอแนะ` — ข้อสรุปและข้อเสนอแนะ (ใช้เป็นเฉลยหลักสำหรับการประเมินนี้ หากไม่มีให้ใช้ข้อ 2 ประกอบ)

ให้ประเมิน OUTPUT โดยเทียบกับ **ข้อ 3 (ข้อสรุปและข้อเสนอแนะ)** ในเฉลย เป็นหลัก

# เกณฑ์การประเมิน (เน้นความตรงประเด็นกับเฉลย)
1. **ความตรงประเด็น (Keyword/Entity Specificity):** 
   - ต้องตรวจสอบว่าข้อเสนอแนะระบุชื่อหน่วยงาน, ขั้นตอนเฉพาะ, หรือคำศัพท์ทางกฎหมาย (Keywords) ตรงกับเฉลยหรือไม่ (เช่น ต้องระบุชื่อสำนักงาน... หรือ ต้องทำหนังสือ...)
   - หากแนะนำกว้างๆ โดยไม่ระบุสิ่งที่เฉลยเน้นย้ำ ให้ถือว่า **"ไม่ครอบคลุม"**
2. **ความสอดคล้องกับเฉลย (Alignment):** 
   - ให้คะแนนตามความสอดคล้องกับเนื้อหาในเฉลยเป็นหลัก มากกว่าความละเอียดของคำแนะนำที่คิดขึ้นเอง
   - แม้คำแนะนำจะดูเป็นไปได้จริง (Practical) แต่หากไม่ตรงกับประเด็นหลักที่เฉลยต้องการสื่อสาร หรือขัดแย้งกับวิธีการในเฉลย ให้ถือว่า **"ไม่ถูกต้องตามเฉลย"**
3. **นโยบายและความเสี่ยง:** 
   - ข้อเสนอแนะต้องสอดคล้องกับธงคำตอบในเฉลยอย่างเคร่งครัด ห้ามแนะนำวิธีเลี่ยงที่เฉลยไม่ได้ระบุไว้

# ผลการประเมิน
- ตอบได้ถูกต้อง ชัดเจน ครอบคลุม ให้ข้อเสนอแนะที่เป็นไปได้จริง (ระบุ Keywords และขั้นตอนตรงตามเฉลย)
- ไม่ถูกต้องตามเฉลย (ขัดแย้งกับเฉลย หรือแนะนำวิธีเลี่ยงที่ไม่ได้รับอนุญาต)
- ตอบได้ถูกต้อง แต่ไม่ครอบคลุม มีเนื้อหาสำคัญตกหล่น (แนะนำกว้างเกินไป ไม่ระบุรายละเอียดเฉพาะเจาะจงตามเฉลย)
