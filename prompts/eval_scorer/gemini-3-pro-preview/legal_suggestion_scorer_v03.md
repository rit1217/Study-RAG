# รูปแบบข้อมูลเฉลย
เฉลย (EXPECTED) อยู่ในรูปแบบ Markdown ที่แบ่งเป็นหัวข้อ:
- `## 1. มาตรากฎหมายที่เกี่ยวข้อง` — มาตรากฎหมายและคำพิพากษาศาลฎีกาที่เกี่ยวข้อง
- `## 2. คำวินิจฉัย` — คำวินิจฉัยที่ถูกต้อง
- `## 3. ข้อสรุปและข้อเสนอแนะ` — ข้อสรุปและข้อเสนอแนะ (ใช้เป็นเฉลยหลักสำหรับการประเมินนี้ หากไม่มีให้ใช้ข้อ 2 ประกอบ)

ให้ประเมิน OUTPUT โดยเทียบกับ **ข้อ 3 (ข้อสรุปและข้อเสนอแนะ)** ในเฉลย เป็นหลัก หากเฉลยไม่มีข้อ 3 ให้ประเมินว่าข้อสรุปใน OUTPUT สอดคล้องกับคำวินิจฉัยในข้อ 2 ของเฉลยหรือไม่

# เกณฑ์การประเมิน
- ตรวจสอบว่าข้อสรุปสอดคล้องกับคำวินิจฉัยและเฉลยหรือไม่
- ตรวจสอบว่าครอบคลุมข้อควรระวังและความเสี่ยงทางกฎหมายที่สำคัญหรือไม่
- **เน้นความเฉพาะเจาะจง (Specificity):** ตรวจสอบว่าข้อเสนอแนะระบุหน่วยงาน ขั้นตอน หรือระเบียบปฏิบัติที่เฉพาะเจาะจงตรงตามเฉลยหรือไม่ หากเฉลยระบุชื่อหน่วยงานเฉพาะ (เช่น สำนักงานอัยการฯ) แต่คำตอบให้คำแนะนำกว้างๆ (Generic advice) ให้ถือว่าไม่ครอบคลุม
- **ความสอดคล้องกับระเบียบปฏิบัติ (Compliance):** ตรวจสอบว่าข้อเสนอแนะสอดคล้องกับระเบียบปฏิบัติของธนาคารตามที่เฉลยระบุไว้หรือไม่ การแนะนำที่ทำได้ทั่วไปแต่ขัดต่อนโยบายธนาคารให้ถือว่าไม่ถูกต้อง

# ผลการประเมิน
- ตอบได้ถูกต้อง ชัดเจน ครอบคลุม ให้ข้อเสนอแนะที่เป็นไปได้จริง
- ไม่ถูกต้องตามเฉลย
- ตอบได้ถูกต้อง แต่ไม่ครอบคลุม มีเนื้อหาสำคัญตกหล่น
