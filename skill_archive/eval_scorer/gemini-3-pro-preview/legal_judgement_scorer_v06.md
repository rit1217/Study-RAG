# งานของคุณ

# รูปแบบข้อมูลเฉลย
เฉลย (EXPECTED) อยู่ในรูปแบบ Markdown ที่แบ่งเป็นหัวข้อ:
- `## 1. มาตรากฎหมายที่เกี่ยวข้อง` — มาตรากฎหมายและคำพิพากษาศาลฎีกาที่เกี่ยวข้อง
- `## 2. คำวินิจฉัย` — คำวินิจฉัยที่ถูกต้อง (ใช้เป็นเฉลยหลักสำหรับการประเมินนี้)
- `## 3. ข้อสรุปและข้อเสนอแนะ` — ข้อสรุปและข้อเสนอแนะ

ให้ประเมิน OUTPUT โดยเทียบกับ **ข้อ 2 (คำวินิจฉัย)** ในเฉลย เป็นหลัก และใช้ข้อ 1 ประกอบเพื่อตรวจสอบการปรับบทกฎหมาย

# เกณฑ์การประเมิน
1. **เหตุผลทางกฎหมาย (Legal Reasoning) เป็นหลัก:**
   - ต้องตรวจสอบว่าการปรับบทกฎหมายเข้ากับข้อเท็จจริงนั้นถูกต้องตามตรรกะทางกฎหมาย ไม่ใช่แค่มีการอ้างเลขมาตราไว้เฉยๆ
   - **สำคัญ:** หากธงคำตอบ (สรุปผล) ถูกต้อง แต่เหตุผลประกอบ หรือหลักกฎหมายที่ใช้อ้างอิงผิด (เช่น อ้างผิดมาตรา, ใช้หลักกฎหมายผิดประเภท) ให้ถือว่า **ไม่ถูกต้องตามเฉลย (0.0)**

2. **ความยืดหยุ่นของผลลัพธ์:**
   - หากผลลัพธ์ไม่ตรงกับเฉลย 100% แต่มีเหตุผลทางกฎหมายที่ยอมรับได้หรือมีการตีความที่สอดคล้องกับบริบท ให้พิจารณาให้คะแนนบางส่วน (0.5) แทนที่จะตัดเป็น 0 ทันที (ยกเว้นกรณีขัดแย้งกับนโยบายธนาคารอย่างร้ายแรง)

3. **ความถูกต้องของผลทางกฎหมาย:**
   - ตรวจสอบสถานะทางกฎหมาย (สมบูรณ์/โมฆะ/โมฆียะ) ว่าสอดคล้องกับเหตุผลที่ยกมาหรือไม่

# ผลการประเมิน
- ไม่ถูกต้องตามเฉลย (ธงผิด หรือ ธงถูกแต่เหตุผล/หลักกฎหมายผิด หรือ มีการยกคำพิพิากษาฎีกาที่ไม่เกี่ยวข้อง หรือไม่มีอยู่จริง หรือหากมีการวินิจฉัยในมุมหรือกรณีที่ไม่เกี่ยวข้องกับคำถาม)
- ตอบได้ถูกต้อง แต่ไม่ครอบคลุม มีเนื้อหาสำคัญตกหล่น (เหตุผลถูกต้องแต่อาจขาดรายละเอียด หรือธงคลาดเคลื่อนเล็กน้อยแต่มีเหตุผลรองรับที่ดี)
- ตอบได้ถูกต้อง ชัดเจน ครอบคลุม (ธงถูก และเหตุผลทางกฎหมายถูกต้องสมบูรณ์)
